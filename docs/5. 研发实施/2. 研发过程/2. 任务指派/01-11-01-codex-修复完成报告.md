## 📋 详细修改清单

### 1. 工具可用性修复（必做）

**文件**: `Dockerfile:42`

**修改前**:
```dockerfile
RUN apk add --no-cache libc6-compat ca-certificates bubblewrap
```

**修改后**:
```dockerfile
RUN apk add --no-cache libc6-compat ca-certificates bubblewrap bash ripgrep
```

**原因**:
- Bash tool 需要 `/bin/bash`（Alpine 默认只有 `/bin/sh → busybox`）
- Grep tool 使用 ripgrep (`rg`)，不是 `/bin/grep`
- SDK 内置的 ripgrep 可能与 Alpine musl libc 不兼容

**验收**:
```bash

docker-compose build

# 验证工具存在
docker exec ex0-app which bash
docker exec ex0-app which rg

# 测试 Bash tool
# 在 Claude Agent 中输入: bash echo "test"
# 应该正常输出: test
```

### 2. Glob 结果为空修复（必做）

**文件**: `ws-query-worker.mjs:101-153`

**修改内容**: 扩展 `workspaceInstructions`，添加三种路径说明：

```javascript
const workspaceInstructions = `

IMPORTANT - File Access and Path Boundaries:

You have access to THREE distinct locations:

1. **Session Workspace** (Read/Write, Relative Paths)
   - Location: ${config.cwd}
   - Use for: Creating, editing, and managing user files
   - Path style: Relative paths only (e.g., "index.html", "src/App.jsx")

2. **Project Source Code** (Read-Only, Absolute Path)
   - Location: /app
   - Use for: Reading framework code, dependencies, and system files
   - Path style: Must use path="/app/..." parameter
   - WARNING: This directory is READ-ONLY. Do not attempt to write here.

3. **User Skills** (Read-Only, via CLAUDE_HOME)
   - Location: ${process.env.CLAUDE_HOME}/.claude/skills/
   - Use for: Loading user-defined custom skills
   - Automatically loaded via settingSources: ['project']

TOOL USAGE GUIDELINES:

**Glob Tool** (file search):
- To search workspace: Use glob("pattern") with relative pattern
- To search project source: Use glob("pattern", { path: "/app" })
...
`;
```

**原因**:
- Glob 默认搜索 workspace（会话工作目录），初始为空
- 用户需要知道如何指定 `/app` 路径搜索项目源码
- 明确区分三种路径类型，避免混淆

**验收**:
```bash
# 测试 Glob（workspace）
# 输入: glob("**/*.ts")
# 预期: 搜索当前 workspace

# 测试 Glob（project source）
# 输入: glob("**/*.ts", { path: "/app/src" })
# 预期: 搜索项目源码目录
```
