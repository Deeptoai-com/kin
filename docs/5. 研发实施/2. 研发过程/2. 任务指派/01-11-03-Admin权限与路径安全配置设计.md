# Admin 权限与路径安全配置设计

**目的**：为组织管理员提供可视化配置入口，管理 Claude Agent 的权限模式与路径安全规则，替代仅靠环境变量配置的方式。

---

## 1. 范围与目标

**范围**：
- 组织级（Organization）配置
- 生效对象：该组织下所有用户与会话

**目标**：
- 可视化设置 `permissionMode` 与 `Bash` 权限
- 可视化配置路径安全追加规则（读/写允许前缀、阻止前缀）
- 变更可审计、可回滚、可提示风险

**非目标**：
- 不在本阶段提供 per-user 或 per-session 级别的路径规则
- 不在本阶段提供 `bypassPermissions` 的强制边界（仍需 sandbox 或 hooks）

---

## 2. 现状（基线）

- 运行时安全规则由 `createPathSecurity()` 生成（默认 allowlist + blocked prefixes）。
- `permissionMode` 和 `allowBash` 当前可从 `organization.metadata` 读取（`server/permissions.server.ts`）。
- 读/写/阻止前缀仅支持环境变量：
  - `CLAUDE_READ_ALLOWED_PREFIXES`
  - `CLAUDE_WRITE_ALLOWED_PREFIXES`
  - `CLAUDE_BLOCKED_PREFIXES`
  - `CLAUDE_SESSIONS_ROOT`

---

## 3. 配置项设计

### 3.1 可配置项（Admin UI）

- 权限模式：`default` / `bypassPermissions`
- Bash 开关：仅在 `bypassPermissions` 时可开启
- 读允许前缀（追加）：`readAllowedPrefixes`
- 写允许前缀（追加）：`writeAllowedPrefixes`
- 阻止前缀（追加）：`blockedPrefixes`

### 3.2 不可配置项（固定规则）

- `/app/` 只读（永远不允许写入）
- `workspace` 与 `${userRoot}/sessions/` 必须保留
- 系统阻止前缀固定存在（`/etc`, `/proc`, `/sys`, `/root`, `/var`, `/bin`, `/usr`, `/sbin`, `/boot`, `/lib`）
- `bypassPermissions` 不提供路径级强制边界

---

## 4. 数据模型（组织 metadata）

**结构建议**（兼容现有字段）：
```json
{
  "permissionMode": "default",
  "allowBash": false,
  "pathSecurity": {
    "readAllowedPrefixes": ["/mnt/shared"],
    "writeAllowedPrefixes": ["/mnt/shared/project"],
    "blockedPrefixes": ["/mnt/secret"]
  }
}
```

**说明**：
- `permissionMode` / `allowBash` 保持现有字段
- `pathSecurity` 作为新字段追加
- 未配置时使用环境变量默认值

---

## 5. 接入路径（服务端）

### 5.1 读取

- 扩展 `getPermissionInfo`：
  - 读取 `organization.metadata.pathSecurity`
  - 合并环境变量默认值

### 5.2 更新

- 扩展 `updateOrganizationPermissions`：
  - 允许更新 `pathSecurity.readAllowedPrefixes`
  - 允许更新 `pathSecurity.writeAllowedPrefixes`
  - 允许更新 `pathSecurity.blockedPrefixes`

### 5.3 生效

- 各入口在调用 `createPathSecurity()` 时传入：
  - `extraReadPrefixes`
  - `extraWritePrefixes`
  - `extraBlockedPrefixes`

**说明**：生效策略建议为“组织配置覆盖环境变量默认值”，无需重启。

---

## 6. Admin UI 设计

### 6.1 页面位置
- Organization Settings → Security / Permissions

### 6.2 区块与表单

1) **权限模式**
- 单选：Standard (default) / Bypass (danger)
- 展示说明：Bypass 模式无权限确认，且不保证路径边界

2) **Bash 权限**
- Toggle（仅在 Bypass 时可开启）

3) **路径安全**
- Read Allowed Prefixes（tag 输入）
- Write Allowed Prefixes（tag 输入）
- Blocked Prefixes（tag 输入）

4) **规则预览**
- 展示 “基础规则 + 追加规则” 合并结果
- 标注 `/app/` 只读、系统阻止前缀不可移除

### 6.3 校验规则

- 仅允许绝对路径（以 `/` 开头）
- 自动去重、规范化（`path.posix.normalize`）
- 不允许 `..` 片段
- `writeAllowedPrefixes` 不允许落在 `blockedPrefixes` 下
- `blockedPrefixes` 的优先级高于允许前缀

---

## 7. 安全与审计

- 仅 `owner/admin` 可编辑
- 保存时记录：
  - 操作人
  - 时间
  - 变更前后差异
- 管理台展示最近一次变更记录

---

## 8. 兼容性与回退

- 没有组织或未配置时，继续使用环境变量
- 若 `pathSecurity` 配置解析失败，回退环境变量默认值
