# Claude 模块重构计划

## 背景与目标

### 问题描述
当前 Claude Chat 相关代码分散在多个目录中，与 Mastra 模块的集中式结构形成鲜明对比：

- Mastra：集中在 `src/mastra/` 下，结构清晰
- Claude Chat：分散在 `components/`、`server/`、`lib/`、`routes/` 等 5 个位置

### 当前代码分布

| 层级 | 路径 | 文件数 | 职责 |
|------|------|--------|------|
| UI 组件 | `src/components/claude-chat/` | 15 | Session 列表、Artifacts、面板等 |
| UI 组件 | `src/components/skills/` | 5 | Skills 卡片、列表、详情 |
| 服务端核心 | `src/server/agent-session.ts` | 1 | AgentSession 类 |
| 服务端核心 | `src/server/agent-session-manager.ts` | 1 | AgentSessionManager |
| 服务端核心 | `src/server/session-title.ts` | 1 | 标题生成 |
| 服务端核心 | `src/server/ws-server.ts` | 1 | WebSocket 服务器 |
| 服务端核心 | `src/server/ws-bootstrap.ts` | 1 | WS 启动逻辑 |
| 服务端核心 | `src/server/skills/` | 5 | Skills 管理器 |
| 适配器 | `src/lib/claude-agent-ws-adapter.ts` | 1 | WS 适配器 |
| 路由 | `src/routes/agents/claude-chat/` | 1 | 页面路由 |
| API 路由 | `src/routes/api/skills/` | 4 | Skills REST API |

### 重构目标

1. 创建 `src/claude/` 模块，与 `src/mastra/` 形成对称架构
2. 集中 Claude Agent 核心业务逻辑
3. 保持 UI 组件和路由在原位置（遵循 TanStack Start 约定）
4. 提高代码可维护性和模块边界清晰度

## 实施步骤

### Phase 1: 创建目录结构
1. 创建 `src/claude/` 目录及子目录
2. 创建 `index.ts` 入口文件

### Phase 2: 迁移 Agent 核心文件
1. 迁移 `agent-session.ts` → `claude/agent/session.ts`
2. 迁移 `agent-session-manager.ts` → `claude/agent/session-manager.ts`
3. 迁移 `session-title.ts` → `claude/agent/title-generator.ts`

### Phase 3: 迁移 Skills 模块
1. 迁移 `server/skills/` 下所有文件 → `claude/skills/`
2. 创建 `claude/skills/index.ts` 统一导出

### Phase 4: 迁移 WebSocket 模块
1. 迁移 `ws-server.ts` → `claude/ws/server.ts`
2. 迁移 `ws-bootstrap.ts` → `claude/ws/bootstrap.ts`

### Phase 5: 迁移适配器
1. 迁移 `lib/claude-agent-ws-adapter.ts` → `claude/adapters/ws-adapter.ts`

### Phase 6: 更新 Import 路径
1. 更新所有引用迁移文件的 import 语句
2. 使用 `~/claude/` alias

### Phase 7: 清理与验证
1. 删除原文件
2. 运行 TypeScript 类型检查
3. 运行开发服务器验证功能

## 风险与回滚

### 风险点
1. Import 路径遗漏导致运行时错误
2. 循环依赖问题

### 回滚策略
- 所有改动在单独 commit 中完成
- 出现问题可 `git revert` 回滚

## 预期收益

1. **架构对称**：`src/claude/` 与 `src/mastra/` 形成清晰对称
2. **模块边界**：Claude Agent 核心逻辑有明确边界
3. **可维护性**：修改 Claude 功能时，核心代码集中在一处
4. **团队协作**：不同团队成员可独立负责不同模块
