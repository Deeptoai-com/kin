# 自定义 Skills 用户指南

**更新时间**: 2025-01-10
**适用版本**: v1.0+

---

## 什么是 Skills？

Skills 是可复用的 AI 技能包，包含：
- **SKILL.md** - 技能定义文件（必需）
- **代码文件** - TypeScript/JavaScript 实现
- **配置文件** - 配置和设置
- **文档** - 使用说明和示例

---

## 快速开始

### 1. 访问技能管理页面

导航到 **代理设置 → Skills**，你会看到：

- **官方 Skills** - 平台提供的官方技能
- **我的 Skills** - 你上传的自定义技能

### 2. 上传新技能

点击 **"上传新技能"** 按钮，进入上传页面。

### 3. 填写技能元数据

#### 必填字段：
- **技能名称** - 唯一标识符（例如：`my-custom-skill`）
  - 最多 50 个字符
  - 只能包含字母、数字、连字符
  - 示例：`code-reviewer`、`document-generator`

#### 可选字段：
- **描述** - 简短说明技能功能
- **分类** - 选择技能类型：
  - `development` - 开发工具
  - `productivity` - 效率工具
  - `design` - 设计辅助
  - `integration` - 第三方集成

### 4. 添加技能文件

#### 必需文件：SKILL.md

每个技能必须包含 `SKILL.md` 文件，格式如下：

```markdown
---
name: My Custom Skill
description: A brief description of what this skill does
category: productivity
---

# My Custom Skill

Provide detailed instructions for Claude here.

## Usage

When to use this skill and how it works.

## Examples

Example usage scenarios.
```

**Frontmatter 字段说明**：
- `name` - 技能名称（必须与上传时的名称一致）
- `description` - 技能描述
- `category` - 分类（development/productivity/design/integration）

#### 可选文件：

你可以添加任意数量的代码文件、配置文件和文档：

```
skill-root/
├── SKILL.md              # 必需：技能定义
├── src/
│   ├── utils.ts         # 工具函数
│   └── config.ts        # 配置
├── README.md            # 使用文档
└── examples/
    └── example.md       # 示例
```

**支持的文件类型**：
- Markdown (.md)
- TypeScript (.ts, .tsx)
- JavaScript (.js, .jsx)
- JSON (.json)
- YAML (.yaml, .yml)
- 文本文件 (.txt)

### 5. 上传和启用

点击 **"上传技能"** 按钮：

- ✅ 上传成功后，技能会**自动启用**
- ✅ 技能立即可用
- ✅ 存储在你的私有空间（`.claude/skills/user/{name}/`）
- ✅ 其他用户无法访问

## 管理自定义技能

### 查看技能列表

在 **"我的 Skills"** 部分，你会看到：
- 技能名称和描述

点击 **"启用"** 或 **"禁用"** 按钮切换状态：

- **禁用** - Claude 不会使用该技能

⚠️ **注意**：删除操作无法撤销

## 资源限制

为了保证系统性能和稳定性，每个技能有：

| 限制项 | 上限 |
|--------|------|
| 文件数量 | 100 个文件 |
| 总大小 | 10 MB |
| 文件名长度 | 255 字符 |
| 技能名称长度 | 50 字符 |

**超出限制时**：
- 上传会失败
- 会显示错误提示
- 请减少文件数量或压缩内容

## 安全建议

### ✅ 安全实践

1. **只上传自己编写或审查过的代码**
   - 你完全理解代码的功能
   - 代码来源可信

2. **使用沙盒环境测试**
   - 先在测试环境验证
   - 确认无异常后再使用

3. **定期审查技能**
   - 删除不再使用的技能
   - 更新有安全问题的代码

### ⚠️ 风险提示

1. **代码执行风险**
   - Skills 会在你的环境中运行
   - 恶意代码可能访问你的数据
   - 只上传可信来源的代码

2. **隐私风险**
   - 技能会处理你的对话内容
   - 不要在技能中硬编码敏感信息
   - 使用环境变量存储密钥

3. **性能影响**
   - 复杂技能可能影响响应速度
   - 资源限制保护系统稳定性

## 常见问题

### Q: 上传后需要手动启用吗？

**A**: 不需要。上传成功后会自动启用，立即可用。你可以随时禁用。

### Q: 可以修改已上传的技能吗？

**A**: 目前不支持在线编辑。如需修改：
1. 删除现有技能
2. 重新上传修改后的版本

### Q: 其他用户能看到我的技能吗？

**A**: 不能。技能存储在你的私有空间，完全隔离。

### Q: 技能文件会被共享到云端吗？

**A**: 不会。所有技能文件都存储在本地 Docker 容器中。

### Q: 如何分享技能给其他用户？

**A**:
1. 导出技能文件（打包为 zip）
2. 其他用户下载后上传到他们的环境
3. 每个用户都有独立的副本

### Q: 上传失败的常见原因？

**A**:
- **文件数量超过 100 个** - 减少文件数量
- **总大小超过 10 MB** - 压缩文件内容
- **文件路径包含 `..`** - 移除相对路径
- **技能名称过长** - 限制在 50 字符以内
- **缺少 SKILL.md** - 确保包含必需文件

### Q: 技能存储在哪里？

**A**:
- 路径：`.claude/skills/user/{skill-name}/`
- 通过 Docker volume 持久化
- 重启容器后仍然保留
